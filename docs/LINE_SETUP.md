## 設計思想

広告 → LP → LINE登録 **までは獲得した後、LINE内で以下の4段階で商談化する**：

```
登録 → ガイド受領 → 物件リクエスト → 相談予約 → 商談
```

各段階の離脱を最小化するため、**自動化できる部分は全てテンプレで仕掛ける**。有人対応は「相談予約」以降に集中させる。

## Phase 7: 一斉配信スケジュール（月次運用）

ステップ配信が流れ終わった後の継続タッチポイント。

### 7.1 月間配信カレンダー

| 頻度 | 曜日 | 時刻 | 内容 |
|---|---|---|---|
| 月2回 | 第1・第3火曜 | 20:00 | **物件情報アップデート**（最新の非公開物件3〜5件） |
| 月1回 | 第2木曜 | 19:00 | **築古投資コラム**（攻略ガイドの補足記事） |
| 月1回 | 第4日曜 | 10:00 | **施工事例紹介**（Before/After 1件） |

### 7.2 配信の原則

- **売込みは2回に1回以下**：情報提供7：売込み3 の比率
- **季節ネタ**：確定申告（2〜3月）、決算期（9月）は刺さりやすい
- **配信数**: 月4〜6回が上限。これ以上はブロック率上がる

## Phase 8: 計測と改善

### 8.1 KPI

| 指標 | 初期目標 | 計測場所 |
|---|---|---|
| 月間友だち追加数 | 14件（広告シナリオBの想定） | 分析 → 友だち |
| ブロック率 | 10%以下 | 分析 → 友だち |
| メッセージ開封率 | 40%以上 | 分析 → メッセージ |
| クリック率（URL） | 10%以上 | 分析 → メッセージ |
| 物件リクエスト送信率 | 30%以上（友だち追加に対して） | チャット検索「物件リクエスト」 |
| 相談予約率 | 10%以上 | チャット検索「相談」 |

### 8.2 週次チェック

毎週月曜に前週の数字を確認：
- ブロック率が15%超え → あいさつメッセージが重すぎる可能性、軽くする
- 開封率が30%以下 → 配信タイミング or 件名の見直し
- 物件リクエスト率が低い → ステップ配信のCTAを強化

## 付録A: よくある落とし穴

1. **あいさつメッセージが長すぎる**: 3通以内・各200字以内に収める。多すぎると離脱。
2. **リッチメニューの画像解像度不足**: 2500×1686px 推奨。圧縮率を上げすぎない。
3. **ステップ配信の設定漏れ**: 「有効期間」を設定しないと配信されない。公開設定を確認。
4. **タグ管理の手動化**: 友だちが100超えたら Lステップ等を検討。手動では回らない。
5. **配信頻度の上げすぎ**: 週2回超えるとブロック率が急上昇。月4〜6回が適正。
