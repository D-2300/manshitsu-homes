## §4. ⚠️ 非公開物件の取り扱い方針（超重要・ここが本質）

### 実態（ユーザーの本音）
1. **弊社には"非公開物件"は手元に多くはない**（正直なところ少ない）
2. 広告で「非公開物件あります」と誘導して登録してもらうが、実態とは乖離あり
3. 登録後、お客様に「非公開物件ください」と言われても、出せる物件がないことが多い
4. ただし**全く無いわけではない**（継続的には入ってくる）
5. **情報収集目的の同業者**からの問い合わせも一定数ある → 誰にでも情報を出すわけにいかない

### なぜ非公開物件があるのか（ガイドに書いた説明）
売主の**人間的事情**による：
- ご家族の離婚で売却せざるを得ない
- 相続の整理として処分する必要がある
- 経営上の都合で表立った市場に出さずに打診している

これらはプライバシー・心情への配慮が必要だから水面下で動く。必ずしも"お得"とは限らない。

### 業者情報収集へのガード（ガイドに書いた）
> 物件情報は売主様からお預かりした信頼の上に成り立って流通しているものです。そのため弊社では、買う意思の明確なお客様に限定して情報をお渡ししています。情報収集のみを目的とされている場合には対応できない旨、ご理解いただけますと幸いです。

→ これは**フィルタリングの正当化**であり、真剣な投資家だけが残る仕掛け。

### ❌ ガイドに**絶対書かない**こと（真実だが毒になる情報）
以下は段階的開示用に保留：
- 情報流通の順序（銀行→不動産屋→身内→知人→…→ネット公開）
- 「身内が先に見る」「美味しい物件は不動産屋が自分で買って回す」
- 「ネット公開時は必ず最高値スタート」
- 「非公開物件の大半は身内が既に見送った物件」

これらを**一気に告知すると"一生の裏切り"になる**（ユーザー表現）。がん告知の比喩：段階を経て自分で気づかせる必要がある。

### 矛盾の処理ルール
- 広告が完全な嘘ではない（たまに持っている） → 詐欺ではない
- 問題は**期待値のコントロール**
- **登録時点の顧客は信仰を保ったまま** → 関係が深まるにつれ徐々に真実を開示
- 最終的には「ありがとう」と言える状態を目指す

## §5. 段階的開示の設計マップ

非公開物件の実態（§4の❌情報）を、以下の順で段階的に開示する設計：

| タッチポイント | 伝える濃度 | 内容 |
|---|---|---|
| **広告 / LP** | 低 | 「非公開物件情報あり」（事実） |
| **LINE登録直後** | 低 | あいさつメッセージ。条件ヒアリング促し |
| **ガイド（現状）** | 中軽 | 売主の人間的事情／プライバシー／信頼性の話で"種まき"のみ |
| **LINE Day 2〜9** | 中 | 表面利回りで失敗する話・築古見極めのコツ（別の切り口で教育） |
| **LINE Day 14〜30** | 中 | ネット公開物件にも価値がある理由を軽く触れる |
| **個別相談（1対1）** | 高 | 「実は業界構造はこうで…」を対面で開示。信頼関係の上で届く |
| **具体物件検討時** | 最高 | 「この物件は買わない方がいい」と本音で |
| **将来の中級者向け記事** | 最高 | 検索してきた意欲ある読者向けに深掘り |

**原則**：登録直後は信仰を壊さない。関係が深まるにつれ濃度を上げる。

このプロジェクトは**繊細な期待値コントロール**の上に成り立っている。

- 広告：非公開物件で誘導
