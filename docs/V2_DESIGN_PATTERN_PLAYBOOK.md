# V2 デザインパターン プレイブック

> 満室デザインLABO で確立された「**楔＋堀＋Poacher-turned-gamekeeper**」の物語構造を、他プロジェクトに移植するためのフレームワーク・仕様書。

## Part 1: 核となる思考フレームワーク

### 1-1. 楔（Wedge）: 入口メッセージ

**定義**: 顧客の検索意図と完全一致する具体物。LP 上部の H1・広告見出しに使う言葉。

**特徴**:
- 顧客の検索語そのもの、または直接応答する言葉
- 動詞で完結（「〜を教えます」「〜から始まる」）
- 他社が真似しにくい位置情報（エリア・役割）を含む

**例**（満室の場合）:
- H1: 「物件を売る側にいた人間が、裏側を教えます。」
- 楔部分: 「物件」「売る側」「裏側」「教える」
- 検索応答: 「収益物件」検索者に「物件の裏情報を教えてくれる人」として名乗る

### 1-2. 堀（Moat）: 信頼の源泉

**定義**: コピーできない人的資源・組織構造。楔の背後で支える「本当の価値」。

**特徴**:
- 属人的（特定の人間・経歴なしには成立しない）
- 業界内の立ち位置
- 他社が構造上マネできない

**例**（満室の場合）:
- KAI: 元不動産仲介20年 → 内装業へ転身
- KEN: 大工歴20年+、年間30件+施工
- 2人合計40年、下請けに出さない、自社施工

### 1-3. Poacher-turned-gamekeeper（密猟者が森林官に転身）

**最強の信頼装置**。元業界人が、業界を辞めて、今は顧客側に立っている物語。

**4つの必須要素**:
1. **専門性の証明**: 20年等の長期経歴
2. **利害関係の不在**: 今は業界と違う立場にいる
3. **意思の可視化**: 「辞めた」という行動の事実
4. **顧客側に立つ宣言**: 現在の立場が顧客と同じ方向を向いている

**例**:
- 元警察 → セキュリティ会社
- 元ハッカー → サイバーセキュリティ企業
- 元食品品質管理者 → 食の安全活動家
- **元不動産仲介 → 内装屋（満室）**

### パターンA: ポジション提示 Hero

**構造（上から）**:
1. ターゲットバッジ（誰向け）: `◆ 宮城の◯◯オーナー向け`
2. 職歴ラベル: `元◯◯20年 × 職業20年`
3. 顔写真セット（2人）: 金リング + 名前 + 役職
4. H1（言い切り）: `◯◯にいた人間が、裏側を教えます。`
5. サブ（事実のみ）: `20年やって、辞めました。`
6. CRAFTSMANSHIP 見出し（両翼ヘアライン）

8. 40年 行: `──◆ 2人で40年の現場 ◆──`

## Part 4: 移植プロセス（4ステップ）

### Step 1: 楔の言語化

**確認すべきこと**:
- 対象顧客の検索語は何か？
- その検索語に対して「〜を教えます」の動詞で応答できるか？
- 顧客の本音の不安は何か？

**出力フォーマット**:
```
H1: 「[楔の動作]にいた人間が、[隠れた真実]を教えます。」
```

**例のバリエーション**:
- 満室: 「物件を売る側にいた人間が、裏側を教えます。」
- 他業界アイデア（参考）:
  - 保険: 「保険を売る側にいた人間が、本当に要らない保険を教えます。」
  - 結婚式: 「結婚式を売る側にいた人間が、盛らない見積りを教えます。」
  - 転職: 「エージェント側にいた人間が、応募者不利の契約を教えます。」

### Step 2: 堀の棚卸し

**Poacher-turned-gamekeeper チェック**:
- [ ] 20年以上の専門性があるか（最低10年）
- [ ] 元業界を辞めた（or 利害が違う立場にいる）
- [ ] 辞めた理由を語れるか（業界の仕組みに対する違和感）
- [ ] 現在の立場が顧客と同方向か（「売らない側」「儲けない立場」）

**4つ全部 YES なら最強の堀**。1つでも欠けると説得力が落ちる。

### Step 3: ビジュアル適用

**コピペできる要素**:
- Hero 5セクション構造（LP 用）
- Hero 単体（HP 用）
- SectionHeading（両翼ヘアライン）
- 数字プラーク
- コミットピル
- 金リング顔写真

**プロジェクト固有で調整する要素**:
- 数字の種類（年数・件数・%）
- ターゲットバッジの文言
- コミットピルの項目
- ピル枠の色トーン（紫以外の場合）

### Step 4: 物語展開（About ページ）

必須セクション:
1. Hero（大きな顔 + H1 + 数字プラーク + 40年行）
2. KAI's Story（or 業界側の人物のストーリー）
3. KEN's Story（or 技術側の人物のストーリー）
4. Business Model 比較図（中抜きなしの可視化）
5. Philosophy（ダーク背景 + 金強調）
6. Our Approach（3 Q&A、左金線カード）
7. CTA

## Part 7: 代表的な失敗パターン（避けるべき）

1. **楔だけコピーして堀を持たない**: メッセージだけ真似ても信頼が生まれない
2. **Poacher-turned-gamekeeper を偽装**: 嘘は感度の高い顧客に見抜かれる
3. **数字を盛る**: 「30年」「1000件」等、実態と乖離した数字は信頼を失う
4. **ビジュアルだけ模倣**: 金と紫を塗っても、物語構造が無ければ「見た目だけ豪華」な薄いページになる
5. **既存の実証値を壊す**: CPA や CVR で回っているページを、実証前のデザインで上書きする（記憶荘 LP でやってはいけない）
